# Validation rules and messages

## Introduction

The automated validation tool checks a set of defined rules to validate the submitted data. Each rule has a code which you can look up below to explain the problem.

Validation rules generate three outcomes:

1.  [**Error**](#errors): The data is invalid and cannot be submitted for manual validation or to GBIF until it is corrected
2.  [**Warning**](#warnings): The data may be correct, but looks suspicious. It needs to be checked to confirm that it is correct before submission.
3.  [**Alert**](#alerts): This is a special warning for vector distribution. The data needs to be checked to ensure it is correct, but if so, it suggests that the species has been detected in an area where it was previously absent.
